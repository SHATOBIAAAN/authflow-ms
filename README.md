# 🛡️ Authflow MS

**Authflow-ms** — это надежный, защищенный и масштабируемый микросервис аутентификации. Спроектирован с использованием строгой многослойной архитектуры (Clean Architecture) и современных стандартов безопасности. Идеально подходит как foundation для любых Node.js/TypeScript проектов.

---

## 🚀 Технологический стек

- **Runtime:** [Node.js 22](https://nodejs.org/) & [TypeScript](https://www.typescriptlang.org/) (Strict Mode + `exactOptionalPropertyTypes`)
- **HTTP фреймворк:** [Fastify 5](https://fastify.dev/)
- **База данных:** [PostgreSQL 17](https://www.postgresql.org/) & [Drizzle ORM](https://orm.drizzle.team/)
- **Кэш & Rate Limit:** [Redis 7](https://redis.io/)
- **Безопасность:** `bcryptjs` (хеширование паролей), `jsonwebtoken` (сессии), кастомный Rate Limiter
- **Логирование:** [Pino](https://github.com/pinojs/pino) (с маскировкой чувствительных данных)
- **Валидация:** [Zod](https://zod.dev/) (строгая типизация env и request/response payload)

---

## 🎯 Ключевые особенности

- **Безопасные сессии (JWT + Database):** Stateless проверка через JWT, но с возможностью мгновенной инвалидации сессии через БД (хеши токенов). Защита от одновременных логинов (генерация уникальных JTI токенов).
- **UUIDv7:** Самые эффективные и безопасные монотонно-возрастающие идентификаторы для PostgreSQL (реализованы на уровне JS для независимости от расширений базы).
- **Role-Based Access Control (RBAC):** Эндпоинты защищены в соответствии с ролями (например, `/admin` доступен только администраторам).
- **Продвинутый Аудит:** Полное логирование (`auditLog`) попыток входа, регистраций, сбросов пароля (в т.ч. IP и User-Agent).
- **Rate-Limiting:** Защита от DDoS и Brute-Force атак на уровне роутов (не больше 3 попыток логина в минуту).
- **Идеальная архитектура:** Строгое разделение слоёв: `Routes` → `Services` → `Repositories` → `DB`. Никакой бизнес-логики в репозиториях, никакого HTTP в сервисах.

## 🛠️ Переменные окружения (.env)

Создайте файл `.env` в корне проекта на основе Zod-схемы. 
Приложение не запустится при отсутствии хотя бы одной обязательной переменной!

```env
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://auth:password@db:5432/authflow
REDIS_URL=redis://redis:6379
JWT_SECRET=your-super-secret-key-minimum-32-chars
JWT_EXPIRES_IN=15m
BCRYPT_ROUNDS=12
LOG_LEVEL=debug
```

## 🐳 Запуск через Docker (Рекомендуемый)

Проект изначально готов к развертыванию вместе со всей инфраструктурой (App + Postgres + Redis).

### Local Development (Dev-режим с Hot Reloading)
Полностью независимый запуск вместе с базой. Код из `src` монтируется через Volume для моментального обновления без перезагрузки контейнера:
```bash
docker compose -f docker-compose.dev.yml up --build
```

### Production (Боевой релиз)
Многоступенчатая сборка (Multi-stage build) скомпилирует TypeScript в JS:
```bash
make prod
# или напрямую: docker compose up --build -d
```

### Миграции БД
При первом запуске необходимо применить Drizzle-миграции (создать структуру таблиц безопасно):
```bash
make migrate
```

## 📡 API Endpoints

Все маршруты автоматически валидируются при помощи *Zod*. При ошибке валидации пользователи получают читаемый статус `400 Bad Request` с массивом неверных полей.

### Аутентификация
- `POST /auth/register` — Регистрация нового аккаунта (`username`, `password`, опционально `role`). Возвращает `201 Created`.
- `POST /auth/login` — Вход. Выдает уникальный JWT. Существует Rate Limit на 3 попытки в минуту. Предыдущие сессии деактивируются.
- `POST /auth/logout` — Выход. Инвалидация текущей сессии в базе (требует Bearer токен).
- `GET /auth/me` — Получение данных своего профиля.

### Администрирование (Requires `Role: admin`)
- `GET /admin/users` — Пагинированный (через cursor) pull списка зарегистрированных пользователей.
- `GET /admin/audit-log` — Просмотр лога активности по конкретному пользователю или всей системе.

### Сброс пароля
- `POST /password-reset/request` — Запуск восстановления пароля (выдает надежный токен восстановления).
- `POST /password-reset/confirm` — Установка нового пароля и закрытие всех старых активных сессий пользователя в рамках единой транзакции БД.

## 🧪 Тестирование (E2E)

Для проверки всех сценариев в проекте встроен мощный E2E-тест на базе модуля `node:test`. 
Он самостоятельно регистрирует тестового юзера, выполняет вход, проверяет Rate Limiter'ы (обезвреживая попытку брутфорса), верифицирует мидлвары ролей и выходит из сессии.

Для запуска тестов (сначала поднимите проект через `make dev`):
```bash
make test
# или напрямую: npx tsx tests/e2e.test.ts
```

## 🧱 Структура проекта
```text
src/
├── routes/          # HTTP-контракты, валидация входящих запросов по Zod
├── services/        # Чистая бизнес-логика (хеширование, генерация событий сессии)
├── repositories/    # Слой доступа к данным (Drizzle). Никакого SELECT * !
├── middleware/      # Bearer-авторизация и Глобальный Fastify перехватчик ошибок
├── db/              # Drizzle Config, пуллинг PostgreSQL, структура таблиц
├── lib/             # Конфигурация Pino, Zod ENV, Иерархия AppError
└── app.ts           # Точка входа, Healthcheck, Graceful Shutdown
```

> ⚠️ **Важно:** В проекте используется `strict` режим TypeScript и запрещено использование `any`. Все `catch` блоки прозрачно логируются и при необходимости пробрасываются дальше.
